# Zac v5 セキュリティ・プライバシー設計

作成日: 2026-04-27  
基準文書: `Zac_planning_v5.md`

---

## 1. 基本方針

Zacでは「いつ、どこのジムに行くか」がセンシティブ情報である。予定、位置、ホームジム、参加者、顔写真を高リスク情報として扱う。

MVPでは、機能の豊富さよりも公開範囲漏れを防ぐことを優先する。

---

## 2. 高リスク情報

| 情報 | リスク | 方針 |
|---|---|---|
| 現在地 | ストーキング、待ち伏せ | MVPでは正確な現在地共有をしない |
| 予定ジム/時間 | 行動予測、待ち伏せ | 公開範囲を厳格に適用 |
| ホームジム | 生活圏推定 | デフォルト非表示 |
| 参加者 | 人間関係、行動履歴 | 予定の公開範囲に従う |
| 顔写真/動画 | 肖像、未成年情報 | 通報、削除、公開範囲制御 |
| 未成年利用情報 | 安全、法務 | MVPでは18歳以上のみ利用可 |

---

## 3. 公開範囲モデル

| visibility | 表示できるユーザー |
|---|---|
| `public` | ブロック関係がなく、対象が削除/非表示でない全ユーザー |
| `followers` | 作成者をフォローしているユーザー、作成者本人 |
| `participants` | 予定/記録に紐づく参加者、作成者本人 |
| `private` | 作成者本人のみ |

ブロックは公開範囲より優先する。公開コンテンツでも、ブロック関係がある場合は表示しない。

コメントは対象リソースの公開範囲を継承する。コメント単体で対象より広い公開範囲を持たせない。

---

## 4. 認可ルール

### 4.1 作成

| 対象 | 作成できる人 |
|---|---|
| 予定 | ログインユーザー |
| 記録 | ログインユーザー |
| 投稿 | ログインユーザー |
| コメント | ログインユーザー。ただし対象を閲覧できること |
| ジム | 管理者 |
| イベント | 管理者 |
| モデレーション | 管理者 |

### 4.2 更新/削除

| 対象 | 更新/削除できる人 |
|---|---|
| 自分の予定 | 作成者。管理者は非表示/削除相当の運営操作のみ |
| 自分の記録 | 作成者 |
| 自分の投稿 | 作成者 |
| コメント | 作成者または管理者 |
| ジム/イベント | 管理者 |
| 通報 | 管理者がステータス更新 |

## 5. ブロック仕様

ブロックした場合:

- 相互フォローを解除する。
- 相互にプロフィール詳細を制限する。
- 相互にフィードへ表示しない。
- 相互に予定、記録、投稿、コメントを表示しない。
- 相手はブロックしたユーザーの予定へ参加できない。
- 相手はブロックしたユーザーのコンテンツにコメントできない。

固定:

- 既存コメントは通常画面では相互に非表示にする。
- 同じ公開予定に既に参加していた場合、ブロック後は相互の参加者表示とコメント表示を抑制し、新規コメント/参加を禁止する。

MVPでは安全側に倒し、ブロック後は相互の既存コメントも通常画面では非表示にする。

---

## 6. 予定の安全仕様

- 予定公開範囲の初期値は `followers`。
- `public` 予定でも正確な現在地は共有しない。
- 予定の公開範囲は作成後に変更できる。
- 参加者一覧は予定の公開範囲に従う。
- `private` 予定は作成者以外に存在を返さない。
- `participants` 予定は参加者以外に存在を返さない。
- ブロック済みユーザーには予定を表示しない。

---

## 7. 画像/メディア

- 画像はSupabase Storageに保存する。
- ユーザーUGC画像はprivate bucketに保存し、署名付きURLで返す。
- 管理者が承認したジム画像はpublic bucketに保存する。
- 削除済みUGCの画像は通常画面で参照できないようにする。
- 不適切画像は通報対象にする。
- 将来は画像モデレーションを検討する。

---

## 8. 個人情報と削除

退会時の基本方針:

| データ | 方針 |
|---|---|
| メールアドレス | 削除 |
| プロフィール | 削除 |
| 非公開ログ | 削除 |
| 公開投稿 | 通常画面から非表示。必要最小限の匿名化メタデータのみ保持 |
| コメント | 通常画面から非表示。必要最小限の匿名化メタデータのみ保持 |
| 監査ログ | 法務/安全目的で必要最小限保持 |

利用規約・プライバシーポリシーにはこの方針を反映する。

---

## 9. 管理者セキュリティ

- 管理画面はadmin権限必須。
- 管理者操作は `audit_logs` に残す。
- 管理者によるユーザー情報閲覧も監査対象にする。
- 管理者APIは一般APIとpathを分ける。
- 本番データのエクスポート機能はMVPで作らない。

---

## 10. レート制限

MVPで最低限制限する操作:

- ログイン試行
- 投稿作成
- コメント作成
- 通報作成
- フォロー/アンフォロー
- 画像アップロード

---

## 11. セキュリティチェックリスト

- APIはサーバー側で認証を検証している。
- APIはサーバー側で公開範囲を検証している。
- APIはサーバー側でブロック関係を検証している。
- 管理者APIはadmin権限を検証している。
- 削除済みデータを通常APIで返さない。
- 通報中/非表示データをフィードに出さない。
- 退会処理でメール等の個人情報が残り続けない。
- 画像URLから非公開画像が漏れない。
- `.env` や秘密情報をリポジトリに含めない。

---

## 12. 固定事項

- 未成年利用: MVPは18歳以上のみ。
- 退会時の公開投稿/コメント: 通常画面から非表示にし、必要最小限の匿名化メタデータのみ保持。
- 画像URL: ユーザーUGCは署名付きURL。ジム公開画像はpublic URL。
- 管理者2FA: ベータ/本番運用前に必須。
- 初期リリース国/地域: 日本国内を前提に利用規約・プライバシーポリシーを用意する。
