# Zac v5 UX・画面フロー設計

作成日: 2026-04-27  
基準文書: `Zac_planning_v5.md`

---

## 1. UX方針

ZacのMVPは「SNSを眺めるアプリ」ではなく、「次に登る行動が自然に決まるアプリ」として設計する。

最重要導線:

```text
ジムを見る -> 保存する -> 予定を作る -> 記録する -> 投稿/反応 -> 次の予定
```

記録は1分以内に保存できる軽さを優先する。

---

## 2. ボトムナビ

MVPのボトムナビ:

```text
ホーム / 探す / 予定 / 記録 / マイページ
```

投稿作成は独立タブにしない。

投稿導線:

- ホームのFAB
- 記録完了後の「投稿する」
- 予定詳細の「募集・共有する」
- ジム詳細の「このジムについて投稿」

---

## 3. 初回利用フロー

```text
起動
  -> 登録/ログイン
  -> オンボーディング
  -> 主な種目選択
  -> クライミング歴選択
  -> よく行くエリア選択
  -> 興味カテゴリ選択
  -> 初期公開範囲選択
  -> ホーム
```

オンボーディング完了時に、ユーザー設定を作成する。

---

## 4. ホーム

目的: 今日/今週登るきっかけを作る。

タブ:

| タブ | 内容 |
|---|---|
| Following | フォロー中の予定・記録・投稿 |
| Nearby | 近くのジム、投稿、イベント |
| Plans | 自分の予定、誘われた予定、保存ジムのイベント |
| Latest | 新着投稿 |

カード:

| カード | CTA |
|---|---|
| 予定カード | 参加する、コメント、保存 |
| 記録カード | いいね、コメント、自分も記録 |
| 投稿カード | いいね、コメント、保存 |
| ジム更新カード | 詳細、予定に追加 |
| レコメンドカード | 保存、フォロー |

---

## 5. 探す

目的: ジム・イベント・ユーザー・投稿を見つける。

主要要素:

- 検索バー
- エリアフィルタ
- 種目フィルタ
- ジム一覧
- イベント一覧
- 保存済み表示

ジム詳細:

- ジム名
- 画像
- 住所/エリア
- 対応種目
- 営業時間概要
- 保存ボタン
- 予定作成ボタン
- 関連投稿/予定

---

## 6. 予定

目的: 自分と周囲の登る予定を管理する。

画面:

- 予定一覧
- 予定詳細
- 予定作成
- 予定コメント
- 予定完了後の記録作成

予定作成項目:

- タイトル
- 開始/終了日時
- 場所: ジムまたは自由入力
- 種目
- 募集状態
- 公開範囲
- 最大参加人数
- メモ
- 通知

予定詳細のCTA:

- 参加する
- 参加キャンセル
- コメント
- 編集
- キャンセル
- 完了
- 記録する

---

## 7. 記録

目的: 登った履歴を軽く残す。

記録作成の最小項目:

- 日付
- 場所
- 種目
- グレードまたは難易度メモ

任意項目:

- 写真
- 動画
- 登った本数
- 最高グレード
- 体感グレード
- メモ
- 一緒に登った人
- 公開範囲

記録完了後の導線:

```text
保存完了
  -> 投稿する
  -> 写真を追加
  -> 次回予定を作る
  -> ログ詳細へ
```

---

## 8. 投稿

目的: 記録や予定から自然に共有を生む。

投稿種別:

- 単独投稿
- 記録由来投稿
- 予定由来投稿
- ジム由来投稿

投稿作成項目:

- 本文
- 画像
- カテゴリ
- 公開範囲
- 紐づくログ/予定/ジム

---

## 9. マイページ

目的: 自分の活動と設定を管理する。

表示:

- プロフィール
- 投稿
- 記録
- 予定
- 保存ジム
- フォロー/フォロワー

設定:

- プロフィール編集
- 公開範囲設定
- 通知設定
- ブロック管理
- 退会

---

## 10. 重要フロー

### 10.1 ジムから予定

```text
探す
  -> ジム詳細
  -> 保存
  -> このジムで予定を作る
  -> 日時/公開範囲を入力
  -> 予定作成
```

### 10.2 予定から記録

```text
予定一覧
  -> 予定詳細
  -> 完了
  -> 今日の登りを記録する
  -> 記録作成画面
  -> 保存
```

### 10.3 記録から投稿

```text
記録保存
  -> 投稿する
  -> 本文/画像/公開範囲を確認
  -> 投稿
```

### 10.4 通報

```text
対象コンテンツ
  -> メニュー
  -> 通報
  -> カテゴリ選択
  -> 任意コメント
  -> 送信
```

---

## 11. MVP固定事項

- 具体レイアウトは `docs/17_wireframe_ui_spec.md` の入力項目/表示項目を基準に実装する。
- 地図表示はMVPに含めない。
- 予定のカレンダー表示はMVPに含めない。リスト表示を優先する。
- 投稿作成FABはホームのみ表示する。
- 記録の動画アップロードはMVPに含めない。画像のみ対応する。
