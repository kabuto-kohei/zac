# Zac v5 テスト計画書

作成日: 2026-04-27  
基準文書: `Zac_planning_v5.md`

---

## 1. 目的

Zac MVPが、主要ユーザーフロー、公開範囲、ブロック、通報、管理操作を安全に満たしているか確認する。

---

## 2. テスト範囲

| 種類 | 対象 |
|---|---|
| 単体テスト | validation、公開範囲判定、ブロック判定、状態遷移 |
| APIテスト | 認証、CRUD、権限、エラー、ページネーション |
| 結合テスト | API + DB、Auth、Storage、Analytics |
| E2Eテスト | 登録から予定作成、記録、投稿までの主要導線 |
| 管理画面テスト | 通報対応、ジム編集、監査ログ |
| セキュリティテスト | 非公開データ漏れ、ブロック漏れ、管理者API |
| 受け入れテスト | MVP Definition of Done |

---

## 3. 受け入れ条件

### 3.1 登録/オンボーディング

- 新規ユーザーが登録できる。
- 表示名、種目、エリア、興味カテゴリ、初期公開範囲を設定できる。
- オンボーディング完了後にホームへ遷移する。

### 3.2 ジム探索

- ジム一覧を表示できる。
- ジム詳細を表示できる。
- ジムを保存/保存解除できる。
- 保存ジムから予定作成へ遷移できる。

### 3.3 予定

- 予定を作成できる。
- `start_at >= end_at` の予定は作成できない。
- `gym_id` または `place_name` のどちらかが必須。
- 予定に参加/キャンセルできる。
- 予定にコメントできる。
- 予定から記録を作成できる。
- 公開範囲外ユーザーには予定が表示されない。

### 3.4 記録

- 記録を作成できる。
- 予定から作った記録に `session_plan_id` が入る。
- 記録を編集/削除できる。
- 非公開記録は本人以外に表示されない。
- 記録から投稿を作成できる。

### 3.5 投稿/フィード

- 投稿を作成できる。
- 投稿にいいね、コメント、保存ができる。
- フィードに予定、記録、投稿が表示される。
- 削除済み/非表示/公開範囲外/ブロック対象は表示されない。

### 3.6 安全/管理

- ユーザーをブロックできる。
- ブロック後は相互にコンテンツが表示されない。
- 通報を作成できる。
- 管理者が通報を確認し、対応できる。
- 管理者操作が監査ログに残る。

## 4. 重点テストケース

| ID | ケース | 期待結果 |
|---|---|---|
| T-001 | 未ログインで予定/記録/投稿/feed APIをGETする | `401` |
| T-002 | ゲストHTML/RSC payloadに予定/記録/投稿が含まれない | 活動データ文字列が含まれない |
| T-003 | 非公開予定を第三者がGETする | `404` または表示不可 |
| T-004 | ブロック済みユーザーが公開投稿を見る | 表示されない |
| T-003 | 参加者限定予定を非参加者が見る | 表示されない |
| T-004 | 予定から記録を作る | planとlogが紐づく |
| T-005 | 記録から投稿を作る | post.source_typeが設定される |
| T-006 | 管理者以外がadmin APIを呼ぶ | `403` |
| T-007 | 削除済み投稿がフィードに出るか | 出ない |
| T-008 | 通報対応後に監査ログが残る | `audit_logs` に記録される |

## 5. 対応端末

MVP初期:

- User Web: Chrome/Safari/Edge最新版
- 管理画面: Chrome最新版

---

## 6. 自動化方針

- validationと公開範囲判定は単体テスト必須。
- APIの主要CRUDはintegration testを作る。
- E2Eは最初の縦スライスに絞る。
- 管理画面は通報対応だけ先にE2E対象にする。

---

## 7. リリース判定

以下を満たすまで公開ベータへ進まない。

- Must機能の受け入れ条件が通っている。
- 重大な公開範囲漏れがない。
- ブロック/通報/管理画面の最低機能が動く。
- 主要イベント計測が発火している。
- クラッシュ/重大APIエラーが既知で放置されていない。
