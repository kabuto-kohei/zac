# Zac v5 ワイヤーフレーム・UI仕様書

作成日: 2026-04-27  
基準文書: `Zac_planning_v5.md`

---

## 1. 目的

各画面に必要な表示項目、操作、入力制約、空状態、エラー状態を定義する。  
この文書はFigma等で詳細ワイヤーフレームを作る前の実装向けUI仕様として扱う。

---

## 2. 共通UI

### 2.1 ボトムナビ

| タブ | アイコン案 | 遷移先 |
|---|---|---|
| ホーム | home | HOME-01 |
| 探す | search | EXP-01 |
| 予定 | calendar | PLAN-01 |
| 記録 | notebook | LOG-01 |
| マイページ | user | ME-01 |

### 2.2 共通状態

| 状態 | 表示 |
|---|---|
| Loading | skeletonを優先 |
| Empty | 1つの主要CTAを表示 |
| Error | 短い説明と再試行ボタン |
| Unauthorized | ログイン誘導 |
| Forbidden/Not found | 表示できない、または存在しない扱い |

## 3. ホーム

表示項目:

- タブ: Following / Nearby / Plans / Latest
- 予定カード
- 記録カード
- 投稿カード
- ジム更新カード
- 投稿作成FAB

カード操作:

- 参加する
- コメント
- 保存
- いいね
- 詳細へ遷移

空状態:

- Followingが空: ユーザー/ジムを探すCTA
- Plansが空: 予定作成CTA

---

## 4. 探す

表示項目:

- 検索入力
- エリアフィルタ
- 種目フィルタ
- ジム一覧
- イベント一覧

入力制約:

- 検索キーワード: 1〜50文字
- フィルタは複数選択可

操作:

- ジム詳細へ遷移
- 保存/保存解除
- 予定作成へ遷移

---

## 5. ジム詳細

表示項目:

- ジム名
- 画像
- 住所/エリア
- 対応種目
- 営業時間概要
- Webサイト
- 保存状態
- ゲストには関連予定/投稿を表示しない
- ログイン後は予定作成、記録、投稿への活動導線を表示する

主要CTA:

- 保存
- このジムで予定を作る
- このジムについて投稿

エラー:

- 非公開/削除済みジムは表示しない。

---

## 6. 予定作成

入力項目:

| 項目 | 必須 | 制約 |
|---|---:|---|
| タイトル | 必須 | 1〜80文字 |
| 開始日時 | 必須 | 終了日時より前 |
| 終了日時 | 必須 | 開始日時より後 |
| ジム | 条件付き | `placeName` がない場合必須 |
| 自由入力場所 | 条件付き | `gymId` がない場合必須 |
| 種目 | 任意 | disciplinesから選択 |
| 募集状態 | 必須 | open/approval_required/invite_only/closed |
| 公開範囲 | 必須 | public/followers/participants/private |
| 最大参加人数 | 任意 | 1以上 |
| メモ | 任意 | 1000文字以内 |

操作:

- 保存
- 下書き保存
- キャンセル

バリデーション:

- 未入力必須項目がある場合、保存ボタンを非活性。
- `startAt >= endAt` はエラー。
- `gymId` と `placeName` が両方空ならエラー。

---

## 7. 予定詳細

表示項目:

- タイトル
- 日時
- 場所
- 種目
- 募集状態
- 公開範囲
- 参加者
- コメント
- 作成者

操作:

- 参加する
- 参加キャンセル
- コメントする
- 編集
- キャンセル
- 完了
- 記録する
- 通報

権限差分:

- 作成者のみ編集/キャンセル可能。
- 参加者または閲覧可能ユーザーのみコメント可能。

---

## 8. 記録作成

入力項目:

| 項目 | 必須 | 制約 |
|---|---:|---|
| 日付 | 必須 | 未来日は原則不可 |
| ジム | 条件付き | `placeName` がない場合必須 |
| 自由入力場所 | 条件付き | `gymId` がない場合必須 |
| 種目 | 任意 | disciplinesから選択 |
| グレード | 任意 | 50文字以内 |
| 概要 | 任意 | 140文字以内 |
| メモ | 任意 | 2000文字以内 |
| 画像 | 任意 | MVPでは最大4枚を推奨 |
| 公開範囲 | 必須 | user_settingsの初期値 |

操作:

- 保存
- 画像追加
- キャンセル

保存後CTA:

- 投稿する
- 次回予定を作る
- 記録詳細を見る

---

## 9. 投稿作成

入力項目:

| 項目 | 必須 | 制約 |
|---|---:|---|
| 本文 | 必須 | 1〜500文字 |
| 画像 | 任意 | MVPでは最大4枚を推奨 |
| カテゴリ | 任意 | categoriesから選択 |
| 公開範囲 | 必須 | public/followers/participants/private |

操作:

- 投稿
- 下書きはMVPでは不要
- キャンセル

投稿成功後:

- 投稿詳細へ遷移。

---

## 10. 通報

入力項目:

- カテゴリ: 必須
- 補足: 任意、1000文字以内

カテゴリ:

- ハラスメント
- スパム
- 不適切画像
- 危険行為の助長
- 個人情報の晒し
- 著作権侵害
- なりすまし
- その他

送信後:

- 完了メッセージを表示し、元画面へ戻る。

---

## 11. MVP固定事項

- 実装開始時点では本書の画面項目をワイヤーフレーム代替仕様とする。Figmaは後続で作成する。
- カラー、タイポグラフィ、コンポーネントルールはNext.js実装開始時に最小デザイントークンとして定義する。
- 地図表示はMVPに含めない。
- 画像アップロードはMVPでは最大4枚に固定する。
- 動画アップロードはMVPに含めない。
