# Zac v5 非機能要件定義書

作成日: 2026-04-27  
基準文書: `Zac_planning_v5.md`

---

## 1. 目的

Zac MVPの品質条件を定義する。対象は性能、可用性、セキュリティ、保守性、拡張性、バックアップ、対応環境、法務である。

---

## 2. 性能

| 項目 | 要件 |
|---|---|
| API応答 | 通常の一覧/詳細APIはp95 1秒以内を目標 |
| フィード | cursor pagination必須 |
| ジム検索 | pagination必須。将来pg_trgm/PostGISで最適化 |
| 画像 | サムネイル/圧縮を検討。原寸を一覧で直接使わない |
| Web初期表示 | 初期表示で不要な大量APIを呼ばない |

---

## 3. 可用性

| 項目 | 要件 |
|---|---|
| MVP稼働 | 単一リージョンで開始可 |
| 障害時 | P0/P1の切り分け手順を運用計画に持つ |
| DB | 日次バックアップ |
| Storage | 削除/復旧方針を定める |

---

## 4. セキュリティ

| 項目 | 要件 |
|---|---|
| 認証 | 全認証必須APIでBearer Token等を検証 |
| 認可 | 作成者/参加者/管理者権限をサーバー側で判定 |
| 公開範囲 | `visibility` を全UGCに適用 |
| ブロック | 公開範囲より優先して表示抑制 |
| 管理者 | `/v1/admin` はadmin権限必須 |
| 秘密情報 | `.env` をコミットしない。`.env.example` のみ管理 |
| レート制限 | ログイン、投稿、コメント、通報、画像アップロードに適用 |

---

## 5. プライバシー

| 項目 | 要件 |
|---|---|
| 位置情報 | デフォルトOFF |
| 現在地共有 | MVPでは実装しない |
| 予定情報 | 初期公開範囲はfollowers |
| ホームジム | デフォルト非表示 |
| 分析イベント | 本文、メール、正確な現在地を送らない |
| 退会 | 個人情報の削除/匿名化方針を定義する |

---

## 6. 保守性

| 項目 | 要件 |
|---|---|
| 型 | TypeScriptで主要型を管理 |
| API | OpenAPIで仕様管理 |
| DB | migrationで変更管理 |
| ログ | APIエラーと管理者操作を記録 |
| 監視 | Sentry等で例外を検知 |
| テスト | 公開範囲/ブロック/状態遷移は自動テスト必須 |

---

## 7. 拡張性

将来追加を想定するが、MVPでは実装しない。

- ルートDB
- ジムアカウント
- 決済
- 会員証
- 予約
- ランキング
- AIコーチング
- カレンダー連携

DB/APIはこれらを妨げない程度に余白を残すが、MVP実装を重くしない。

---

## 8. 対応環境

| 対象 | 要件 |
|---|---|
| User Web | Chrome/Safari/Edgeの最新版を優先 |
| 管理画面 | Chrome最新版 |
| 言語 | 初期は日本語 |
| タイムゾーン | 日本市場前提でAsia/Tokyoを基本に扱う |

---

## 9. 法務/審査

| 項目 | 要件 |
|---|---|
| UGC | 通報、ブロック、削除、モデレーションを用意 |
| プライバシー | プライバシーポリシーを公開前に用意 |
| 利用規約 | UGC、禁止事項、免責、退会を含める |
| App審査 | 退会導線、通報導線、ブロック導線を確認 |
| 課金 | MVPでは実装しない。開始時は別途法務確認 |

---

## 10. 固定事項

- SLA目標: MVPでは数値SLAを外部公開しない。内部運用はP0即時、P1当日、P2次回リリース候補、P3通常バックログ。
- 画像の保存期間と削除タイミング: 削除済みUGCの画像は通常APIで返さず、非同期削除対象にする。
- 退会時の公開投稿/コメント: 通常画面から非表示にし、必要最小限の匿名化メタデータのみ保持。
- 未成年利用: MVPは18歳以上のみ。
- 管理者2FA: ベータ/本番運用前に必須。
